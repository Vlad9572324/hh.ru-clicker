# Структура серверной части HH

`web_app.py` остаётся точкой входа для совместимости. Логика серверной части разделена по зонам ответственности:

- `state.py` - общая рабочая конфигурация и список аккаунтов.
- `storage.py` - сохранение JSON, кеши в памяти и вспомогательные функции доступа к данным.
- `hh_client.py` - HTTP-запросы к HH.ru, парсинг, анкеты, вспомогательные функции для резюме и LLM.
- `bot.py` - рабочее состояние аккаунтов, менеджер WebSocket-подключений и цикл бота.
- `routes.py` - FastAPI-приложение, WebSocket-точка подключения, обработчики REST API и цикл рассылки состояния.

Файлы интерфейса разделены похожим образом:

- `static/index.html` - HTML-разметка.
- `static/css/dashboard.css` - стили панели.
- `static/js/dashboard.js` - клиентская логика панели.
