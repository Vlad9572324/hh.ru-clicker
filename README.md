# HH Bot Dashboard

Веб-интерфейс для автоматизации работы с hh.ru: массовые отклики, авто-ответы
работодателям через LLM, заполнение опросников, аудит резюме, мониторинг
переговоров. Работает в реальном времени через WebSocket.

---

## Что умеет

- **Массовые отклики** — параллельный сбор вакансий по нескольким поисковым
  URL, фильтры по зарплате/графику/опыту, контроль дневного лимита HH
- **LLM авто-ответы** — DeepSeek / OpenAI / Anthropic (любой OpenAI-совместимый
  API) пишет персонализированные ответы HR с учётом резюме и истории чата
- **Заполнение опросников** — нейронка отвечает на вопросы работодателя
  при отклике; шаблоны как запасной вариант
- **Робот-рекрутер** — детектится по сигнатурам, бот жмёт кнопку «Да»/«Согласен»
  вместо обычного ответа
- **Аудит резюме** — конкурентность по запросам (вакансий / соискателей),
  скилы конкурентов, проблемы заполнения, горячие лиды
- **Региональный поддомен** — опционально работать через `<region>.hh.ru`
  (Сыктывкар, Москва, Вологда и т.д.)
- **OAuth API** — отклики через официальный API hh.ru вместо web-cookies (опц.)
- **Мониторинг резюме** — просмотры, приглашения, отказы за 7 дней, авто-подъём
  раз в 4 часа
- **Мультиаккаунт** — несколько браузерных сессий параллельно, индивидуальные
  настройки
- **Backup/restore** — выгрузка всего конфига одним файлом и обратная заливка
  с защитой от случайного затирания

## Быстрый старт

### Docker (рекомендуется)

```bash
git clone https://github.com/Vlad9572324/hh.ru-clicker.git
cd hh.ru-clicker
docker-compose up -d
```

→ открыть http://localhost:8000

### Локально (Python)

```bash
git clone https://github.com/Vlad9572324/hh.ru-clicker.git
cd hh.ru-clicker
pip install -r requirements.txt
python web_app.py
```

→ открыть http://localhost:8000

### Доступ из локальной сети

По умолчанию сервер слушает только `127.0.0.1`. Чтобы открыть с других
устройств в LAN:

```bash
HH_BOT_HOST=0.0.0.0 HH_BOT_UNSAFE_EXPOSE=1 python web_app.py
```

WebSocket принимает same-origin запросы автоматически — никаких
дополнительных настроек не нужно.

### Безопасность

Если бот будет доступен из сети, обязательно ставь API-ключ:

```bash
HH_BOT_API_KEY=<secret> python web_app.py
```

Тогда требуется `?key=<secret>` в URL или `X-API-Key` в заголовках.

## Первый запуск

1. **Добавить сессию из браузера**: Настройки → Браузерные сессии →
   «➕ Добавить сессию из браузера». Открой hh.ru в браузере, скопируй
   любой запрос как cURL (F12 → Network → правая кнопка → Copy as cURL),
   вставь в форму. Бот достанет cookies, проверит сессию, найдёт твои
   резюме.

2. **Настроить поисковые URL**: Настройки → 🔗 Пул поисковых запросов.
   Добавь URL поиска (можно скопировать со страницы hh.ru/search) или
   воспользуйся «✨ Подсказки по резюме» — бот сам предложит запросы
   из аудита твоего резюме с показом конкуренции по каждому.

3. **(Опционально) Настроить LLM**: Настройки → 🤖 LLM Авто-ответы →
   «➕ Добавить профиль». Заполни `api_key`, `base_url`, `model`. Для
   DeepSeek: `https://api.deepseek.com` + `deepseek-v4-flash`. Включи
   чекбокс «Автоотправка» если хочешь чтобы бот сам слал ответы (без
   него — будут только черновики).

4. **Запустить аккаунт**: На главной нажми **▶ Запустить** на карточке
   сессии. Бот начнёт собирать вакансии и отправлять отклики.

## Конфигурация

Все настройки хранятся в `data/config.json` (создаётся автоматически).
Можно редактировать через GUI (Настройки → JSON-редактор) или текстом.

### Ключевые переменные окружения

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| `HH_BOT_HOST` | `127.0.0.1` | На каком интерфейсе слушать |
| `HH_BOT_PORT` | `8000` | Порт |
| `HH_BOT_UNSAFE_EXPOSE` | (пусто) | `=1` чтобы разрешить host вне loopback |
| `HH_BOT_API_KEY` | (пусто) | API-ключ для авторизации REST/WS |
| `HH_BOT_ALLOWED_ORIGINS` | (пусто) | Доп. хосты в WS Origin whitelist |
| `HH_CHATIK_BASE` | `https://chatik.hh.ru` | Chatik base URL (есть allowlist) |
| `LLM_PROXY` | (пусто) | Прокси **только** для запросов к LLM (см. ниже) |

### Прокси для LLM (`LLM_PROXY`)

Если сервер с российским IP не может достучаться до LLM-провайдера
(OpenAI и т.п.), запросы к нему можно завернуть в прокси, **не трогая
hh.ru-трафик** (он по-прежнему идёт напрямую — это важно, потому что hh.ru
обычно наоборот хочет видеть РФ-IP).

```bash
# http/https или socks5 (для socks нужен пакет httpx[socks])
LLM_PROXY="http://user:pass@1.2.3.4:8080"
# LLM_PROXY="socks5://user:pass@1.2.3.4:1080"
```

В Docker — добавьте в `environment:` сервиса `hh-bot` (`docker-compose.yml`):

```yaml
    environment:
      LLM_PROXY: "http://user:pass@1.2.3.4:8080"
```

Через прокси идут и реальные чат-вызовы, и тест-коннект «Проверить ключ».
В отличие от глобального `HTTPS_PROXY`, `LLM_PROXY` **не** заворачивает
запросы к hh.ru. Реализация: [`app/llm.py`](app/llm.py) (`_make_openai_client`)
и [`app/routes/llm.py`](app/routes/llm.py) (`_llm_proxies`).

### Региональный поддомен

`config.hh_region` (через UI или JSON):

- пусто → `https://hh.ru` для всех запросов (по умолчанию)
- `syktyvkar` → `https://syktyvkar.hh.ru` для поиска, откликов, переговоров,
  резюме, аудита
- OAuth (`/oauth/...`) и chatik (`chatik.hh.ru`) **всегда** на основном
  домене (HH так требует)

## Технологии

| Компонент | Стек |
|-----------|------|
| Backend | Python 3.10+, FastAPI, uvicorn |
| Frontend | Vanilla JS + WebSocket (живой апдейт 300мс) |
| LLM | OpenAI-совместимый API (DeepSeek / OpenAI / Anthropic / OpenRouter / …) |
| HH API | `requests` + `aiohttp` для async-сбора, SSR-парсинг JSON из HTML |
| Чат | `chatik.hh.ru/chatik/api/` (reverse-engineered) |
| OAuth | hh.ru OAuth2 (Android-клиент credentials) |
| Тесты | pytest, 130+ unit-тестов |
| Деплой | Docker + docker-compose |

## Требования

- **Python 3.10+** (для локального запуска без Docker)
- **Активная сессия hh.ru** — нужны cookies `hhtoken`, `_xsrf`, `crypted_id`,
  `hhul` (бот сам достаёт `hhuid` / `crypted_hhuid` при первом запросе)
- **API-ключ LLM-провайдера** — опционально, только для авто-ответов и
  заполнения опросников нейросетью

---

## Безопасность и приватность

- `data/` в `.gitignore` — личные cookies и API-ключи не попадают в репо
- Бэкап содержит реальные cookies/ключи — храни как пароль, не публикуй
- API-ключ HH (`HH_BOT_API_KEY`) защищает дашборд от чужого доступа
- WebSocket Origin-проверка отбрасывает CSWSH-атаки
- CSP-заголовки на статике, redact-логи для отладки
